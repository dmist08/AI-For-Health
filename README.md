# AI for Health - SRIP 2026

## Project Overview
This project detects sleep apnea events from Nasal Airflow, Thoracic Movement, and SpO2 signals. The pipeline applies precise DSP filtering, windows signals into 30-second segments, and uses a Multi-Branch 1D CNN optimized for severe class imbalance.

## Folder Structure
Ensure you place the `internship/Data` files provided inside the directory as they are messy. Run `setup_data.py` to organize them into the required structure automatically. 

```
AI_for_Health/
├── Dataset/                   # Output folder for raw CSV & PKL
├── Data/                      # Standardized data generated by setup_data.py
├── Visualizations/            # Output folder for visual checks like AP01 PDF
├── results/                   # Evaluation artifacts (confusion matrices, csv)
├── setup_data.py              # 1. Organization script
├── scripts/
│   ├── parsers.py             # Parses all messy formats
│   ├── vis.py                 # 2. Generates timeline
│   ├── create_dataset.py      # 3. Features & Windowing pipeline
│   └── train_model.py         # 4. Neural Network LOPO CV Pipeline
├── models/
│   └── cnn_model.py           # MultiBranchBreathingCNN Architecture
└── src/
    ├── config.py              # Hardcoded hyperparameters
    └── utils.py               # Seeds and Logger
```

## Running the Pipeline

1. **Format Base Data structure**: 
    ```bash
    python setup_data.py
    ```
2. **Visualize & Validate** a participant (Check `Visualizations/`):
    ```bash
    python scripts/vis.py -name Data/AP01
    ```
3. **Generate ML Datasets** (Outputs `breathing_dataset.csv` and `.pkl`):
    ```bash
    python scripts/create_dataset.py
    ```
4. **Train and Cross-Validate** (LOPO):
    ```bash
    python scripts/train_model.py
    ```

## 🔴 Critical Notes & Edge Cases

*   **Mixed Apnea Filtering**: "Mixed Apnea" events are merged into "Obstructive Apnea" to prevent model over-focusing on extreme N=2 samples as requested via review.
*   **Filter Logic**: SpO2 is **not** bandpass filtered per specification to maintain latency indicators.
